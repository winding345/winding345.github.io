Hugo使用于个人blog的搭建

April 18, 2019

1 个人blog的结构

Hugo作为一个用于静态网页的生成打包及发布的工具，在制作个人blog上有着较多的应用。而个人blog一般有以下几个内容组成：

  • 网页的框架——基本的网站交互结构，如http协议等，此部分内容由Hugo工具自动生成，一键式搭建网页框架；
  • 网页的元素与布局——一个网页的可视化内容，类似于QQ空间的装扮设置，此部分内容可以自己编写相应的xml等前端配置文件生成主题或直接采用第三方设置好的主题(类似于QQ空间中自己设置的装扮或者是QQ空间给你提供的装扮套装)
  • 网页的内容——你的博文和介绍等，如QQ空间的日志和说说

接下来本文将从以上三个方面来介绍如何使用Hugo来搭建个人blog。

2 blog的搭建过程

2.1网页的框架的构建

设置网页的框架是Hugo的基本功能。Hugo可以通过一条命令直接完成一个网页框架的搭建。 Hugo的基本操作一般由终端命令来完成，虽然在Windows环境下Hugo的部分操作仍然可以通过直接在资源管理器中对文件进行修改与移动，但是不推荐使用，尽可能使用终端命令有助于对终端相关内容的学习与理解。

  • 首先打开终端,进入你存储个人blog内容的文件夹，如下列示例代码中的”D:\hugo”，在该文件夹下输入以下命令并回车：

hugo new site yoursitename

  • 创建完毕后，将会在该目录下创建出一个相应的文件夹，如图所示。 由图中可以看出，hugo在创建完成一个网页框架后，生成了一下几个内容：

archetypes文件夹——网页结构样式文件夹，用于存放网页结构的文件夹，不自定义主题的情况下可以无视

config.toml文件 ——网页配置文件，对网页中模块的增删中主要需要修改的文件

content文件夹——内容文件夹，用于存放网页中的博文等内容，自己写的文章都存放与此文件夹中

data文件夹——暂时不明确

layouts文件夹——网页结构层次文件夹，不自定义主题的情况下可以无视

static文件夹——用于存放网页中一些元素内容的文件夹，如图片等

themes文件夹——存放主题的文件夹

  • 通过以上命令操作，一个网页的基本框架已经构建成功，但想将此网页在浏览器中进行展示，仍然需要相应的主题来对其进行可视化

2.2 主题的设置

  • 前文已经提到，主题可以有两种方式来得到，一是自己写相关的配置文件来自定义主题，二是自行悬着第三方提供的主题，下文仅介绍如何安装第三方的主题。第三方主题一般是在hugo官方网站中选择确认后，通过链接跳转至github，再在github中进行clone至2.1节中提到的themes文件夹中，即完成了下载。以下以ananke主题(主题的github地址点我)为例显示相应的下载示意。

  • 主题下载完成后，注意主题的文件夹名，如图例中下载的主题文件夹名为gohugo-theme-ananke，这是在配置文件中使用的主题名称。打开上述文件中的config.toml文件(记事本可直接打开)，追加以下代码，完成主题设置，若使用其他主题，只需修改为对应的主题名称即可.

theme=gohugo-theme-ananke

  • 主题设置完毕后，网页已经可以完成对外的显示，退出主题文件夹，在网页的主目录下进行本地部署，部署命令如下：

hugo server

  • 打开本地浏览器，输入以下代码或者直接点我进行访问网页,网页显示如下图所示：

https://localhost:1313

2.3 内容添加

由于未对博客添加内容，所以导致了网页中只有着基本的主题框架，而框架中没有任何内容，因此，接下来需要对网页的内容进行相应地添加。

  • 首先对主页进行相应地美化。在content文件夹下创建文件_index.md(_index.md文件是网页的相关设置文件，content目录下为主页的配置文件，而在content目录下级的各个目录中的_index.md文件为各个分页面的配置文件)，可以在终端中使用以下命令完成创建：

hugo new _index.md

创建完成后，对_index.md文件进行编辑，以以下内容替换原内容：

# 和python类似，#为注释，所有的内容可以继续对应修改，观察网页的相应变化，了解各个参数的作用。

# 在网页进行部署的状态中可以直接修改本文件，修改保存后网页将自动实时刷新，便于观察相应的变化。预览的可能有一定的刷新问题，发现不对的话可以关掉本地部署，再部署一次

# 以下各个属性均不是必须内容，可以人为删减，不进行设置

--- # 这里是三个横杠，下同

title: “这是标题” #标题名称，显示在网页上

date: 2019-04-17T15:47:02+08:00 # 日期，可以省略

description: “这是一个描述” # 主页中添加相应的描述

featured_image: “./images/index.jpg” # 这是背景图片，图片存放于static目录下，填写图片的目录地址，省略static及之前的部分

---

#在此之后为主页显示的正文，不属于配置部分

若出现网页中报错，有格式乱码的问题，需要进行相应的格式转换，（注意在整个网页设计中，所有的文本md的编码必须为utf-8)，文本的编码转换可以直接通过windows的记事本完成。

用记事本打开文件，选择左上角文件——另存为，在下方的编码处选择UTF-8格式，原文件名称覆盖保存即可，如下图所示：

  • 博客网页不能简单的只有一个主页，因此给博文添加标签分页。分页设置需要在配置文件config.toml中添加一行配置：

SectionPagesMenu = “main”

添加完成后，在content文件夹下每一个有md文件的文件夹都将生成一个标签页。以下是添加一个post标签，并添加blog文章的实例。

  • 首先要在content目录下创建一个文件夹post，并在其中创建_index.md文件(标签页的配置文件)，在终端输入图中命令完成文件夹与文件的创建：

修改刚刚创建的_index.md文件，进行相应的配置，配置文件的内容与主页的_index.md相似，此次使用的内容只设置名称为post。

  • 接下来在post标签页中添加我们的主题内容blog文章。创建blog的方式与创建_index.md的方式相同，代码如下：

hugo new post/first.md

创建完毕后，对md进行修改，进行blog文章编写吧！blog的文件可以按照markdown的格式进行编写，[我的下一篇blog]()将简单介绍常见的markdown语法，也可以直接使用文件进行编辑。如有需要抬头和编写时间，文本的开头要加上一下配置信息：

---

Title : 文章名

Author : 作者名

date : 时间

---
